# Business Tools Service — API Documentation

Welcome to the Business Tools Service API. This service provides AI-agent capabilities (chat streaming, async background processing) designed for multi-tenant SaaS environments. 

All endpoints are secured using a robust **Zero-Trust HMAC Signature** strategy to prevent unauthorized access and replay attacks.

---

## Authentication (HMAC SHA-256)

Every request to the `business-tools-service` must include three required headers. The signature is generated by hashing the current timestamp and the raw JSON payload body using your `App Secret`.

### Required Headers
| Header | Description |
|--------|-------------|
| `X-App-Id` | Your public Application ID (e.g., `app_c8df99680e63c5de`) |
| `X-Timestamp` | Current Unix timestamp in seconds (e.g., `1718386348.123`) |
| `X-Signature` | The computed HMAC-SHA256 hash |

> [!WARNING]
> Requests with a timestamp older than **60 seconds** will be rejected as a replay attack. Ensure your server clocks are synchronized via NTP.

## Endpoints

### 1. Enqueue Agent Job (Async)
`POST /v1/agent/enqueue`

Accepts an AI agent job and returns immediately. The service processes the job in the background and delivers the final result via a webhook callback to your specified URL.

**Request Payload:**
```json
{
  "job_uuid": "123e4567-e89b-12d3-a456-426614174000",
  "apiKey": "sk-proj-your-llm-api-key",
  "provider": "openai",
  "model": "gpt-4o",
  "userPrompt": "Analyze the lead data and summarize it.",
  "systemPrompt": "You are a helpful AI assistant.",
  "callback": {
    "url": "https://your-domain.com/api/webhook/ai-result",
    "method": "POST"
  },
  "tools": ["crm_read", "crm_write"],
  "context": {
    "tenant_id": 5
  }
}
```

**Response (200 OK):**
```json
{
  "status": "accepted",
  "job_id": "123e4567-e89b-12d3-a456-426614174000"
}
```

### 2. Stream Agent Chat (SSE)
`POST /v1/agent/chat`

Initiates a Server-Sent Events (SSE) stream for real-time frontend chat interactions.

**Request Payload:**
```json
{
  "job_uuid": "chat-session-999",
  "apiKey": "sk-proj-your-llm-api-key",
  "userPrompt": "Hello! What can you do?",
  "use_stream": true,
  "history": [
    {"role": "user", "content": "Hi"}
  ]
}
```

**Response (200 OK - `text/event-stream`):**
```text
data: {"type": "content", "data": "Hello! I can "}
data: {"type": "content", "data": "help you manage leads."}
data: {"type": "end"}
```

## Code Examples

### 1. Node.js (Robust Implementation using `crypto`)

Use this industry-standard snippet to securely generate the signature and make the request.

```javascript
const crypto = require('crypto');

/**
 * Generates the HMAC authentication headers for the Business Tools Service.
 * 
 * @param {string} appId - Your App ID
 * @param {string} secret - Your App Secret
 * @param {object|string} payload - The JSON payload to sign
 * @returns {object} The required HTTP headers
 */
function generateHmacHeaders(appId, secret, payload) {
    // 1. Generate current Unix timestamp in seconds
    const timestamp = (Date.now() / 1000).toString();
    
    // 2. Ensure payload is a string exactly as it will be sent
    const payloadString = typeof payload === 'string' ? payload : JSON.stringify(payload);
    
    // 3. Concatenate timestamp and payload
    const message = timestamp + payloadString;
    
    // 4. Generate SHA-256 HMAC
    const signature = crypto
        .createHmac('sha256', secret)
        .update(message, 'utf8')
        .digest('hex');
        
    return {
        'X-App-Id': appId,
        'X-Timestamp': timestamp,
        'X-Signature': signature,
        'Content-Type': 'application/json'
    };
}

// === Usage Example ===

async function enqueueJob() {
    const APP_ID = 'app_c8df99680e63c5de';
    const SECRET = 'sk_live_31d6451f8c8cc626d481d567e7faaa08';
    const ENDPOINT = 'http://localhost:8017/v1/agent/enqueue';

    const payload = {
        job_uuid: crypto.randomUUID(),
        apiKey: "sk-proj-...",
        userPrompt: "Process my request asynchronously.",
        callback: {
            url: "https://my-domain.com/webhook",
            method: "POST"
        }
    };

    const payloadString = JSON.stringify(payload);
    const headers = generateHmacHeaders(APP_ID, SECRET, payloadString);

    try {
        const response = await fetch(ENDPOINT, {
            method: 'POST',
            headers: headers,
            body: payloadString
        });

        const data = await response.json();
        console.log("Response:", response.status, data);
    } catch (error) {
        console.error("Request Failed:", error);
    }
}

enqueueJob();
```

### 2. cURL (Bash Script)

If you need to test the API directly from the terminal, you can use the following bash script to compute the signature and execute the `curl` command.

```bash
#!/bin/bash

APP_ID="app_c8df99680e63c5de"
SECRET="sk_live_31d6451f8c8cc626d481d567e7faaa08"
ENDPOINT="http://localhost:8017/v1/agent/enqueue"

# The raw JSON payload
PAYLOAD='{"job_uuid":"test-job-1","apiKey":"sk-proj-...","userPrompt":"Hello","callback":{"url":"https://hook.com"}}'

# 1. Generate timestamp (seconds.milliseconds)
TIMESTAMP=$(date +%s.%3N)

# 2. Concatenate timestamp and payload
MESSAGE="${TIMESTAMP}${PAYLOAD}"

# 3. Compute HMAC SHA-256 signature
SIGNATURE=$(echo -n "$MESSAGE" | openssl dgst -sha256 -hmac "$SECRET" | awk '{print $2}')

# 4. Execute curl
curl -X POST "$ENDPOINT" \
     -H "Content-Type: application/json" \
     -H "X-App-Id: $APP_ID" \
     -H "X-Timestamp: $TIMESTAMP" \
     -H "X-Signature: $SIGNATURE" \
     -d "$PAYLOAD"
```
